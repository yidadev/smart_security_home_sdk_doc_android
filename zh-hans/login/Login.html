2 用户服务

2.1 用户服务同步

通过 1.1.x 的接口注册成功或者登录成功后，需要校验user的服务状态。立即调用getServiceInfoWithHandler来检查服务绑定的情况；

用户服务相关的两种情况：

1.使用安防SaaS默认绑定：安防SaaS在后台默认配置了一个服务商给客户，所有的用户信息会全部收集在这个SaaS平台上，并且不区分；

2.不做安防SaaS默认绑定: 需要自己登录安防SaaS后台创建激活码，通过getServiceInfoWithHandler返回的数据model，校验是否绑定，如果没有绑定，然后通过bindServiceWithCode;这种方式，可以通过激活码来区分用户的渠道。

接口说明

获取并同步用户的服务信息

获取用户绑定的服务商Id，渠道ID，绑定的激活码，并且可以判断出客户是否绑定激活码

如果不需要激活码的操作，直接联系 Tuya 开发开启默认激活码，这样保证每次获取到的值都是绑定过的

void getServiceInfoWithHandler(ITuyaResultCallback<ServiceDealerBean> callback);

参数说明：

参数 说明
callback 成功回调

返回值说明：

返回值 类型 说明
isBind boolean 是否绑定激活码
code String 激活码
channelId String 绑定的激活码对应的渠道ID
dealerId String 绑定的激活码对应的服务商ID

代码示例：

ITuyaSecurityServiceSdk mTuyaSecurityService = TuyaOptimusSdk.getManager(ITuyaSecurityServiceSdk.class);
final ITuyaSecurityService iTuyaSecurityService = mTuyaSecurityService.newServiceInstance();
            iTuyaSecurityService.getServiceInfoWithHandler(new ITuyaResultCallback<ServiceDealerBean>() {
                @Override
                public void onSuccess(ServiceDealerBean result) {

                }

                @Override
                public void onError(String errorCode, String errorMessage) {

                }
            });

2.2 绑定激活码

接口说明：

客户可以根据自己创建的激活码，对于用户进行渠道区分

void bindServiceWithCode(String code, ITuyaResultCallback<Boolean> callback);

参数说明：

参数 说明
code 激活码
callback 回调

代码示例：

ITuyaSecurityServiceSdk mTuyaSecurityService = TuyaOptimusSdk.getManager(ITuyaSecurityServiceSdk.class);
final ITuyaSecurityService iTuyaSecurityService = mTuyaSecurityService.newServiceInstance();
iTuyaSecurityService.bindServiceWithCode("your code", new ITuyaResultCallback<Boolean>() {
                @Override
                public void onSuccess(Boolean result) {

                }

                @Override
                public void onError(String errorCode, String errorMessage) {

                }
            });
